INTRODUCTION

Semantic HTML, also known as semantic markup, refers to the use of HTML tags that conveys the meaning of the content contained within them.

By adding semantic HTML tags to your pages, you provide additional information that helps define the roles and relative importance to the different parts of the page.

TAGS

There are multiple tags that can be used to make your HTML code more semantic.

Tag Code Description
Main 
<main> </main>

The main attribute is used to encapsulate the dominant content within a webpage.
Footer 
<footer> </footer>

The footer attribute is located at the bottom of the page an usually contains information such as contact information, copyright information, terms of use, site map, reference to tope of page links
Header 
<header> </header>

The header attribute usually contrains the <h1> to <h6> headings or the <nav> tag is it is being used for block navigation for the page.
Nav 
<nav> </nav>

This tag is used to define a block of navigation links such as menus and tables of contents.
Section 
<section> </section>

This tag defines elements in a document, such as chapters, headings, or any other area of the documetn with the same theme.
Article 
<article> </article>

This tag holds content that makes sense on its own.It can hold content such as articles, blogs, comments, magazines, ect.

Figure 
<figure> </figure>

This tag is used to encapsulate media such as an imge, illustration, diagram, code snipet, ect. Which is referenced in the main flow of the document.
Figure Caption 
<figcaption> </figcation>

This tag is added to <figure> element to describe the image.
Audio 
<audio> </audio>

This tag is used to embed audio content into a document.
Video 
<video> </video>

This tag makes it clear that the developer is attempting to display a video to the user.
Embed 
<embed/>

This tag can embed any media content including videos, audio files, and gifs from an external source.

Embed is a self closing tag. It is also a deprecated tag and other alternative should be used in its place.

TIPS & BEST PRACTICES

Dont' Use Semantic HTML Tags for Styling

Although web browsers apply styling to many smenatic tags (e.g., the text within an <a> tag is usually blue and underlined), it doesn't mean HTML tas are supposed to be used to style your text.

Don't use an <a> tag for a "non-link" text just to make it blue and underlined, you should not sue other semantic tags for purely sylistic purposes.

Here are some typlical examples of using semantic tags incorrectly:

  • Using a 
    <h1>
    to 
    <h6>
    tag for text that is not a heading with the purpose of changing its font size
  • Using 
    <blockquote>
    just to indent text that is not a quotation
  • Using 
    <strong>
    or 
    <em>
    just to add bold or italics to text that doesn't need emphasis

For styling purposes, always use CSS

Nest the Heading Tags with Semantics in Mind

Always order header elements by importance.

This way, the structure of headings creates a logical hierarchy of topics with your pieve of content and helps both readers and search engines better understand and navigate the text.

Don't Just Copy Your Visual Layout

Your HTML implementation shouldn't be a mere duplicate of the visual layout. Instead, it should follow the semantic structure of the page.
